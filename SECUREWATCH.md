## рҹҸ  SIEM Nima? вҖ” Oddiy Tushuntirish

**SIEM** (Security Information and Event Management) вҖ” bu sizning serverlar, kompyuterlar va tarmoq qurilmalarini **real vaqtda kuzatib turadigan** va hujumlarni **avtomatik aniqlaydiagn** tizim.

### Uy xavfsizligi bilan taqqoslash

Tasavvur qiling: sizda katta bir ombor bor. Omborda:
- 50 ta xona bor, har birida eshik bor
- Har kecha kimdir kirishga urinishi mumkin
- Siz hamma xonani bir vaqtda ko'ra olmaysiz

**Oddiy yechim:** Har bir eshikka qo'riqchi qo'yish вҶ’ Juda qimmat, 50 ta qo'riqchi kerak.

**SIEM yechimi:**
```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ                                                             в”Ӯ
в”Ӯ   Har bir eshikka вҶ’ Kamera (Agent)                         в”Ӯ
в”Ӯ   Barcha kameralar вҶ’ Markaziy monitor (SIEM Server)        в”Ӯ
в”Ӯ   Monitor вҶ’ Avtomatik tahlil (Detection Engine)            в”Ӯ
в”Ӯ   Tahlil вҶ’ Xabar + Harakat (Alert + Active Response)       в”Ӯ
в”Ӯ                                                             в”Ӯ
в”Ӯ   1 ta opertor 50 ta xonani bir vaqtda kuzatadi!           в”Ӯ
в”Ӯ                                                             в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

### Kiberxavfsizlikda SIEM

```
Ombor = Sizning kompyuter tizimingiz (serverlar, PK'lar)
Eshik  = Login sahifalari, SSH, API, portlar
Kamera = Agent (har bir serverda o'rnatiladi)
Xabar  = Log (har bir hodisa qayd etiladi)
SIEM   = Barcha loglarni tahlil qiluvchi markaziy miya

Har bir harakatni kuzatadi:
  вң“ Kim kirdi?        вҶ’ auth.log
  вң“ Nima o'zgardi?    вҶ’ file integrity monitoring
  вң“ Qayerdan keldi?   вҶ’ IP geolokatsiya
  вң“ Qachon bo'ldi?    вҶ’ timestamp
  вң“ Xavfli emasmi?    вҶ’ 36 ta qoida tekshiruvi
```

### Har bir komponent nimР° qiladi?

**NGINX** вҖ” "Darvozabon"
- Barcha trafikni qabul qiladi va to'g'ri joyga yo'naltiradi
- Sekundiga 100+ so'rov kelib qolsa вҖ” sekinlashtiradi (rate limiting)
- Zararli so'rovlarni bloklaydi (`.git`, `.env` fayllarni himoya qiladi)

**Backend (FastAPI)** вҖ” "Miya"
- Barcha mantiqni bajaradi
- Agentlardan ma'lumot qabul qiladi
- Qoidalarni tekshiradi
- Alertlar yaratadi
- Active Response ishga tushiradi

**Frontend (React)** вҖ” "Ko'z"
- Operator ko'radigan panel
- Real vaqtda alertlar ko'rsatadi (WebSocket)
- Grafalar, jadvallar, qidiruv

**PostgreSQL** вҖ” "Asosiy xotira"
- Doimiy ma'lumotlar: foydalanuvchilar, qoidalar, alertlar
- O'chib qolmaydi, disk'da saqlanadi

**Elasticsearch** вҖ” "Log arxivi"
- Barcha loglar shu yerda saqlanadi
- Millionlab logdan bir sekundda qidiruv
- 90 kundan keyin avtomatik o'chirish

**Redis** вҖ” "Tez xotira"
- Sessiyalar (kim tizimga kirgan)
- Rate limit hisoblagichlari
- Bloklangan IP ro'yxati
- WebSocket xabarlarini ko'p serverga tarqatish

```

Internet uzildi:

## рҹ“Ҳ Anomaliya Deteksiyasi

### Oddiy tushuntirish

Tasavvur qiling: Siz kuniga o'rtacha 100 ta email olasiz.
Bugun 10,000 ta email keldi вҶ’ Bu anomaliya!

SIEM ham xuddi shunday. Har bir server uchun "normal" ko'rsatkichlarni o'rganadi va ulardan keskin farq qilsa вҶ’ alert.

### Welford Algoritmi (Statistika)

```
Muammo: Million ta log uchun o'rtacha va standart og'ishni hisoblash
Oddiy usul: Barcha qiymatlarni xotirada saqlash вҶ’ Impossibly katta RAM

Welford yechimi: Faqat 3 ta son zarur
  n    вҶ’ Nechta hodisa ko'rildi
  mean вҶ’ Joriy o'rtacha
  M2   вҶ’ Dispersiya uchun yig'ma

Yangi qiymat kelganda (O(1) operatsiya):
  delta = new_value - mean
  mean  = mean + delta / n
  M2    = M2 + delta * (new_value - mean)
  var   = M2 / (n - 1)
  std   = sqrt(var)

Redis'da saqlash: {n, mean, M2} вҶ’ Bitta agent uchun bitta kalit
```

### Z-Score Hisoblash

```
Z-score = (joriy_qiymat - o'rtacha) / standart_og'ish

Misol:
  web-server-01 uchun login urinishlari:
  O'rtacha: 12/soat
  Std:       3.2

  Bugun:    450/soat
  Z-score = (450 - 12) / 3.2 = 136.9

  Z > 3.0 вҶ’ HIGH anomaly    (statistik ehtimol: 0.3%)
  Z > 2.0 вҶ’ MEDIUM anomaly  (statistik ehtimol: 5%)

Alert: "web-server-01: login_attempts 136.9Пғ dan yuqori (HIGH)"
```

### Kuzatiladigan metrikalar

```
Har agent uchun:
  login_attempts     вҶ’ SSH/auth login urinishlari soatiga
  failed_logins      вҶ’ Muvaffaqiyatsiz loginlar
  cpu_usage          вҶ’ CPU foizi
  memory_usage       вҶ’ RAM foizi
  network_bytes      вҶ’ Tarqatiladigan ma'lumot hajmi
  unique_ips         вҶ’ Nechta yangi IP manzil ulandДұ
  process_count      вҶ’ Ishlaydigan jarayonlar soni
  dns_queries        вҶ’ DNS so'rovlar soni

  Brute Force himoyasi:
    10 ta noto'g'ri kod вҶ’ Hisob 30 daqiqa bloklanadi

### Qatlam 3: Ma'lumotlar Bazasi Xavfsizligi

```
PostgreSQL:
  вң“ Connection pooling (SQLAlchemy async)
  вң“ Parametrlangan so'rovlar (SQL Injection yo'q)
  вң“ Alohida DB foydalanuvchisi (root emas)
  вң“ Shifrlangan parollar (bcrypt, cost=12)

Elasticsearch:
  вң“ X-Pack Security yoqilgan
  вң“ Basic Auth (username/password)
  вң“ TLS/SSL (ixtiyoriy, sozlanuvchi)
  вң“ Faqat backend'dan kirishimumkin

Redis:
  вң“ AUTH parol himoyasi
  вң“ Faqat localhost va backend'dan kirish
  вң“ Sessionlar shifrlangan JWT

Parollar:
  bcrypt, cost factor 12
  Taxminiy brute force vaqti: 100 yil+
```

      re.search(pattern, text)

  DEBUG=false holda:

  DEBUG=true holda:
    Vaqtinchalik kalit yaratiladi (development uchun)

## вқ“ Tez-tez So'raladigan Savollar

**Q: SIEM qanchalik server ta'sir qiladi?**
A: Agent CPU'ni < 0.5%, RAM'ni < 256MB ishlatadi. Asosiy tizim ishlashiga ta'sir qilmaydi.

**Q: Internet bo'lmasa nima bo'ladi?**
A: Agent SQLite buferga yozadi. Internet qaytganda barcha ma'lumot yuboriladi. Hech narsa yo'qolmaydi.

**Q: Nechta agent qo'shsa bo'ladi?**
A: Cheklov yo'q. 8GB RAM server 100+ agentni boshqara oladi.

**Q: Loglar qancha vaqt saqlanadi?**
A: Default 90 kun. .env'da LOG_RETENTION_DAYS o'zgartirish mumkin.

**Q: HTTPS qanday yoqiladi?**
A: SSL sertifikat oling (Let's Encrypt bepul), nginx.conf'ga qo'shing va HTTPвҶ’HTTPS redirect yoqing.

**Q: Qoidani test qilsa bo'ladimi?**
A: Ha. Rules sahifasida "Test" tugmasi bor. Haqiqiy log matni kiritib qoidani sinab ko'rish mumkin.

**Q: 2FA yo'qolsa nima bo'ladi?**
A: Backup kodlar bilan kirish mumkin. Ular Settings вҶ’ 2FA sahifasida ko'rsatiladi va faqat bir marta ishlatiladi.

**Q: Windows agent Task Scheduler'da ishlaydi, Windows Service emas. Nega?**
A: pywin32 kutubxonasi (Windows Service uchun zarur) turli Windows versiyalarida DLL muammolari keltirib chiqaradi. Task Scheduler tashqi kutubxona talab qilmaydi va barcha Windows versiyalarida ishonchli ishlaydi.

**Q: Elasticsearch'ni alohida serverga ko'chirsam bo'ladimi?**
A: Ha. docker-compose.yml'da elasticsearch servisini o'chiring va backend .env'da ELASTICSEARCH_URL=http://boshqa-server:9200 qo'ying.
